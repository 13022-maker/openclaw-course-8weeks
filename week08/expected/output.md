# 校園助教小幫手輸出（示例）

## TL;DR
- HTTP 定義了 client/server 的請求與回應方式。
- API 常用 JSON 傳資料。
- API Key 不可外洩，錯誤處理要清楚。

## 10 個重點
- GET 用於讀取
- POST 用於建立
- PUT 用於更新
- DELETE 用於刪除
- 200 表成功
- 404 表找不到
- 500 表伺服器錯誤
- JSON 常用於資料交換
- 不要把 API Key 上傳
- 錯誤訊息要可讀
